TTC Bus Tracker

A Raspberry Pi-based transit tracker that pulls TTC bus information and outputs route details to an LCD through breadboard wiring for a specific area and selected stops.

Hardware Feature Built
View GitHub

Overview

This project connects software and hardware by collecting TTC route or stop information and displaying useful bus updates on a physical LCD screen. The current version focuses on a specific area, selected stops, a small set of routes, and nearby connecting subway lines.

Technical Focus

The project highlights Raspberry Pi setup, breadboard wiring, LCD display control, API/data handling, and clean formatting for real-time transit information.

Built Features

  • Collected TTC route or stop information for selected stops in a specific area.
  • Displayed transit information on an LCD module.
  • Connected the LCD to a Raspberry Pi using breadboard wiring.
  • Formatted a small set of bus routes and connecting subway lines for quick physical display output.

Next Steps

  • Test tracker accuracy across the selected stops, routes, and connecting subway lines.
  • Add route calculation from point A to point B for supported areas.
  • Build a digital logger to verify record format, timestamp rollover, ordering, and buffer-full behavior.
  • Add a prediction wrapper test plan using request/response checks, timeout handling, safe defaults, and reference-model comparisons.
  • Run end-to-end traffic scenarios with sensor noise, missing data, route events, logged outputs, and predicted ETAs.
